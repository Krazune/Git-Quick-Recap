Git Quick Recap

A quick recap of Git's features, in HTML format.

Getting help

help - Display detailed help about git.

git help

Display information about git.

git help commit

Display information about the specified git command.

Configuring git

config - Manage the git configurations.

git config --list --show-origin

List configurations, and show their origin file.

git config --global user.name "John Doe"

Set the name of the user on the global configuration file.

git config --global user.email "johndoe@email.com"

Set the e-mail of the user on the global configuration file.

git config --global core.editor emacs

Set the default text editor to use with git on the global configuration file.

Here are the configuration file scopes that can be used (from lowest to highest priority):

  • system (for all users of the system)
  • global (for all repositories of the system user)
  • local (for the repository in which this configuration file exists)

Creating a repository

init - Create a new repository.

git init

Create a new repository in the current directory.

git init /path/to/new/repository/

Create a new repository in the specified path.

git init --bare

Create a new bare repository.

clone - Clone an existent repository.

git clone example.com/url/to/remote/repository

Create a local copy of another repository, in the current directory.

Committing changes

add - Add files to the staging area.

git add example.txt another_example.txt

Add the specified files to the staging area.

git add example_directory

Add all files in the specified directory to the staging area.

git add .

Add all files in the current directory to the staging area.

rm - Remove files from the repository.

git rm example.txt another_example.txt

Delete the specified files from the disk and add their removal to the staging area.

git rm --cached example.txt

Remove the specified files from the staging area without deleting them from the disk.

git rm -r example_directory another_directory

Delete the specified directories from the disk and add their removal to the staging area.

mv - Move files.

git mv /old/path/file.txt /new/path/file.txt

Moves a file from a source path to a destination path and adds the change to the staging area.

commit - Commit the changes in the stage area.

git commit

Commit the changes of the staging area (the text editor will open to choose the commit message).

git commit -m "Commit message"

Commit the changes of the staging area and choose the commit message without using an external text editor.

git commit --amend

Replace the last commit with the new changes in the staging area (the text editor will open to choose the commit message).

git commit --amend --no-edit

Replace the last commit with the new changes in the staging area and keep the same commit message.

Branching and merging

branch - Manage the repository branches.

git branch

List the local branches.

git branch -a

List the local and the remote-tracking branches.

git branch new_branch

Create a new branch at the current commit.

git branch -d example_branch

Delete a branch.

checkout - Switch branches.

git checkout branch_name

Switch to specified branch.

git checkout -b new_branch_name

Create a new branch, and switch to it.

merge - Merge multiple branches.

git merge branch_name

Merge the specified branch into the current branch.

git merge --no-ff branch_name

Merge the specified branch into the current branch, without fast forwarding.

tag - Manage git tags.

git tag

List tags.

git tag v3.5

Tag the current commit.

git tag -a v7.0 -m "Software version 7.0"

Tag the current commit and add a message to it.

stash - Manage the stash.

git stash list

Display the entries on the stash.

git stash

Move the changes to files, and staging area, to the stash.

git stash apply

Apply the most recent entry on the stash.

git stash apply stash@{2}

Apply the specified entry on the stash.

git stash pop

Apply the most recent entry on the stash, and remove it from the stash.

git stash drop

Remove the most recent entry on the stash.

git stash clear

Remove all entries on the stash.

Inspecting the repository

status - Display information about the state of the repository.

git status

Display information about the current directory, and staging area.

log - Display the commit history.

git log

Display the commit history.

git log example_file

Display the commit history of the specified file.

git log --all

Display the commit history of branches, and tags.

git log --graph

Display the commit history in a text-based graph.

git log --oneline

Display the commit history but limit each entry to a single line.

blame - Display the author information of changes.

git blame example_file

Display the author information of every line in the specified file.
